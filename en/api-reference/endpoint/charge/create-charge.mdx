---
title: "Create Charge"
sidebarTitle: "POST /charge"
api: "POST https://api.gates2b.com/charge"
authMethod: "authorization"
description: "Creates a charge. Optionally creates the first payment attempt alongside it (PIX or INTERNATIONAL) in a single request. Standalone attempts can also be created later via POST /charge/{chargeId}/attempts."
---

<Info>
`paymentMethod = CREDIT_CARD` **is not accepted** on this endpoint — sensitive card data must go through the public checkout only (`POST /charge/{id}/checkout/payment` with `X-Checkout-Token`).
</Info>

<Note>
  <Badge color="green">New</Badge> **Payment Split** — distribute the charge amount among multiple recipients using the `split[]` field. Supports fixed value (`FIXED` in BRL) and percentage (`PERCENT` over `netAmount`). See the parameter details below.
</Note>

## Parameters

<ParamField header="Authorization" type="string" placeholder="Bearer {token}" required>
  Bearer authentication token.
</ParamField>

<ParamField body="grossAmount" type="string" placeholder="10.50" required>
  Gross charge amount in BRL (two decimal places).
</ParamField>

<ParamField body="currency" type="string" placeholder="BRL" required>
  Charge currency (ISO 4217). Currently always `BRL` — for payment in another currency, use `checkoutCurrency` with `paymentMethod = INTERNATIONAL`.
</ParamField>

<ParamField body="paymentMethod" type="string" placeholder="PIX">
  If provided, creates the first attempt alongside the charge.
  Accepted values:
  - `PIX` — creates an attempt with PIX instructions (QR code, EMV).
  - `INTERNATIONAL` — creates a checkout session via Revolut. Requires `checkoutCurrency`.

  If omitted, the charge is created as `PENDING` with no attempt.
</ParamField>

<ParamField body="checkoutCurrency" type="string" placeholder="USD">
  Currency in which the payer will pay at the gateway (ISO 4217, e.g. `USD`, `EUR`, `MXN`). **Required** when `paymentMethod = INTERNATIONAL`.
</ParamField>

<ParamField body="description" type="string" placeholder="Test charge">
  Free-text description for the charge.
</ParamField>

<ParamField body="expiresAt" type="string" placeholder="2030-12-31T23:59:59.000Z">
  Expiration date in ISO 8601 format.
</ParamField>

<ParamField body="externalReference" type="string" placeholder="order-123">
  External reference from your system for tracking.
</ParamField>

<ParamField body="attemptIdempotencyKey" type="string" placeholder="uuid-v4">
  Idempotency key for the attempt. Reusing the same key returns 409.
</ParamField>

<ParamField body="webhookUrl" type="string" placeholder="https://your-site.com/webhooks/charge">
  URL called when the charge is paid (POST with payload).
</ParamField>

<ParamField body="gatewayOptions" type="object">
  <Expandable title="Fields">
    <ParamField body="redirectUrl" type="string">Return URL after payment (Revolut).</ParamField>
    <ParamField body="expireAt" type="string">Attempt expiration at the gateway (ISO 8601).</ParamField>
  </Expandable>
</ParamField>

<ParamField body="customerMeta" type="object">
  Customer data. Used by gateways when not overridden at payment time.
  <Expandable title="Fields">
    <ParamField body="name" type="string">Customer name.</ParamField>
    <ParamField body="email" type="string">Customer email.</ParamField>
    <ParamField body="document" type="string">Document (CPF/CNPJ).</ParamField>
    <ParamField body="document_type" type="string">Document type.</ParamField>
    <ParamField body="phone" type="string">Phone number.</ParamField>
    <ParamField body="source" type="string" placeholder="PRE_FILLED">Origin (`PRE_FILLED` or `COLLECT_AT_CHECKOUT`).</ParamField>
  </Expandable>
</ParamField>

<ParamField body="split" type="array">
  <Badge color="green">New</Badge> List of recipients for payment split. Omitting this field creates the charge without split.
  <Expandable title="Fields per item">
    <ParamField body="email" type="string" required>
      Recipient's email registered on the platform.

      <Warning>
        The recipient must have an active account on the platform as a `CUSTOMER` user. Emails not found or belonging to users with a different profile fall as residual to the charge owner — **does not block creation**, but the amount will not be forwarded to the intended recipient.
      </Warning>
    </ParamField>
    <ParamField body="kind" type="string" required>
      Split type:
      - `FIXED` — fixed amount in BRL (provide `amount`). Sum of FIXED items must be ≤ `grossAmount`.
      - `PERCENT` — percentage over `netAmount` (provide `percent`). Sum of PERCENT items must be ≤ 100.
    </ParamField>
    <ParamField body="amount" type="string" placeholder="50.00">
      Fixed amount in BRL (e.g. `"50.00"`). **Required** when `kind = FIXED`.
    </ParamField>
    <ParamField body="percent" type="number" placeholder="10">
      Percentage from 0.01 to 100. **Required** when `kind = PERCENT`.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="id" type="string">
  Without `paymentMethod`: ID of the **charge**.
  With `paymentMethod`: ID of the **attempt** (the charge ID is in `chargeId`).
</ResponseField>

<ResponseField name="chargeId" type="string">
  Charge ID (present only when the response is an attempt).
</ResponseField>

<ResponseField name="status" type="string">
  `PENDING` / `PROCESSING` / `PAID` / `EXPIRED` / `CANCELED` / `FAILED` / `REFUNDED`.
</ResponseField>

<ResponseField name="paymentMethod" type="string">
  `PIX` or `INTERNATIONAL` (present only when there is an attempt).
</ResponseField>

<ResponseField name="grossAmount" type="string">
  Gross charge amount (2 decimal places).
</ResponseField>

<ResponseField name="currency" type="string">
  Charge currency (ISO 4217).
</ResponseField>

<ResponseField name="checkoutMeta" type="object">
  Instructions for the front-end to complete the payment.
  <Expandable title="Fields">
    <ResponseField name="url" type="string">Public checkout URL.</ResponseField>
    <ResponseField name="paymentMethod" type="string">Method (`PIX` / `INTERNATIONAL`).</ResponseField>
    <ResponseField name="qr_image" type="string">Base64 QR image (PIX only).</ResponseField>
    <ResponseField name="qr_text" type="string">PIX BR Code as text (PIX only).</ResponseField>
    <ResponseField name="emv" type="string">PIX EMV (PIX only).</ResponseField>
    <ResponseField name="integrationToken" type="string">Public token to initialize the Revolut Checkout Web SDK (INTERNATIONAL only).</ResponseField>
    <ResponseField name="currency" type="string">Checkout currency (INTERNATIONAL only).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fxMeta" type="object">
  Exchange rate applied (INTERNATIONAL only).
  <Expandable title="Fields">
    <ResponseField name="fromCurrency" type="string">BRL.</ResponseField>
    <ResponseField name="toCurrency" type="string">Target currency (e.g. USD).</ResponseField>
    <ResponseField name="rate" type="number">Applied rate.</ResponseField>
    <ResponseField name="originalAmount" type="number">Original amount in BRL.</ResponseField>
    <ResponseField name="convertedAmount" type="number">Converted amount.</ResponseField>
    <ResponseField name="amountWithSpread" type="number">Final amount charged (with spread).</ResponseField>
    <ResponseField name="amountCents" type="number">Final amount in cents.</ResponseField>
    <ResponseField name="fetchedAt" type="string">Moment of the quote.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="expiresAt" type="string">Expiration date.</ResponseField>
<ResponseField name="description" type="string">Charge description.</ResponseField>
<ResponseField name="externalReference" type="string">External reference.</ResponseField>
<ResponseField name="customerMeta" type="object">Customer data.</ResponseField>

<ResponseField name="splitConfig" type="array">
  Split configuration provided at creation (present when `split[]` was sent).
  <Expandable title="Fields">
    <ResponseField name="email" type="string">Recipient email.</ResponseField>
    <ResponseField name="kind" type="string">`FIXED` or `PERCENT`.</ResponseField>
    <ResponseField name="amount" type="string">Fixed amount in BRL (FIXED).</ResponseField>
    <ResponseField name="percent" type="number">Percentage (PERCENT).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="splitSettlement" type="array">
  Calculated settlement: how much each recipient receives. Rounding remainder (PERCENT) always goes to the owner.
  <Expandable title="Fields">
    <ResponseField name="userId" type="number">User ID on the platform.</ResponseField>
    <ResponseField name="amountCents" type="number">Amount the user will receive (cents).</ResponseField>
    <ResponseField name="isOwner" type="boolean">`true` when the charge owner.</ResponseField>
    <ResponseField name="sourceEmail" type="string">Recipient email.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">Creation date (ISO 8601).</ResponseField>
<ResponseField name="updatedAt" type="string">Last update date (ISO 8601).</ResponseField>

## Create Standalone Attempt

`POST /charge/{chargeId}/attempts`

Creates a payment attempt for an existing charge (server-side, authenticated via JWT). Accepts `PIX` and `INTERNATIONAL`.

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `paymentMethod` | string | Yes | `PIX` or `INTERNATIONAL` |
| `currency` | string | Conditional | Target currency (required for `INTERNATIONAL`, ISO 4217) |
| `redirectUrl` | string | No | Return URL (used for INTERNATIONAL) |
| `attemptIdempotencyKey` | string | No | Idempotency key (recommended) |

<CodeGroup>
```bash PIX
curl -X POST "https://api.gates2b.com/charge/{chargeId}/attempts" \
  -H "Authorization: Bearer {your-token}" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentMethod": "PIX",
    "attemptIdempotencyKey": "uuid-v4"
  }'
```

```bash INTERNATIONAL
curl -X POST "https://api.gates2b.com/charge/{chargeId}/attempts" \
  -H "Authorization: Bearer {your-token}" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentMethod": "INTERNATIONAL",
    "currency": "USD",
    "redirectUrl": "https://your-site.com/return",
    "attemptIdempotencyKey": "uuid-v4"
  }'
```
</CodeGroup>
