# Gates2B API - LLM Integration Reference
# https://gates2b.com
# This file helps AI coding assistants (Cursor, Claude Code, Copilot, etc.)
# integrate with the Gates2B payment processing API.

## Overview

Gates2B is a payment processing platform supporting PIX, Credit Card, and Cryptocurrency payments.
Base URL: https://api.gates2b.com
Crypto Gateway URL: https://api-gateway.gates2b.com
Dashboard: https://painel.gates2b.com

## Authentication

All endpoints require a Bearer token in the Authorization header.
Obtain your API key from the Gates2B dashboard at https://painel.gates2b.com → Integration section.

Header format:
  Authorization: Bearer {your-api-key}

---

## Minimum Transaction Amount

The minimum transaction amount depends on the fee configured for each account.
Default minimum: R$ 3,00 (no account has a fee below this value).
Applies to all payment methods (PIX, Credit Card, Cryptocurrency).
Requests below the account's configured minimum will return HTTP 400.

---

## Webhook Response URL (REQUIRED for transaction status)

To receive transaction status updates (e.g., payment confirmed, denied, refunded),
you MUST configure a Webhook Response URL globally in the dashboard:

  https://painel.gates2b.com → Integration → Webhook URL

This URL is configured ONCE at the account level and is used for ALL transactions
created through the API (PIX, Card, Crypto). It is not sent per-request.

Without a configured webhook URL, your system will not be notified when a
transaction changes status — you would have to poll the invoice endpoints instead.

Webhook URL requirements:
  - Must be publicly accessible (HTTPS recommended).
  - Must respond with HTTP 2xx within a reasonable timeout to acknowledge receipt.
  - Must handle retries idempotently (Gates2B retries up to 3 times on failure).

Example webhook URL: https://your-domain.com/webhooks/gates2b

## Endpoints

### 1. Validate API Key

GET /api-key/validate

Validates whether your API key is active and returns expiration info.

Response 200:
{
  "status": 200,
  "code": "string",
  "response": {
    "valid": true,
    "message": "string",
    "expires_at": "string | null"
  }
}

Response 401: Invalid or expired key.

---

### 2. Create PIX Payment

POST /payments

Creates a PIX payment and returns a QR code for the customer.

Request body (JSON):
{
  "clientMode": "noNeedToSpecify | fillDataNow | requestCheckout",  // required
  "amount": 100.00,                    // required, number
  "expire_at": "2026-12-31T23:59:59Z", // required, ISO 8601
  "description": "Order #123",         // optional
  "external_reference": "REF-001",     // optional
  "redirect_url": "https://...",       // optional, post-payment redirect
  "phoneNumber": "+5511999999999",     // optional
  "email": "customer@example.com",     // optional
  "document_type": "CPF | CNPJ",      // optional
  "name_client": "John Doe",          // optional
  "document_client": "12345678900"     // optional
}

Response 201:
{
  "id": 12345,
  "amount": 100.00,
  "status": "string",
  "qr_image": "base64-encoded-png",
  "qr_text": "string",
  "emv": "string",
  "checkout_id": "string",
  "createdAt": "ISO 8601 (America/Sao_Paulo)",
  "updatedAt": "ISO 8601 (America/Sao_Paulo)"
}

Errors: 400 (invalid data), 401 (unauthorized), 500 (server error).

---

### 3. Create Credit Card Payment

POST /payments/v1/card

Creates a credit card payment and returns a secure checkout URL.

Request body (JSON):
{
  "clientMode": "string",       // required
  "amount": 250.00,             // required, number
  "expire_at": "ISO 8601",      // required
  "installments": 3             // required, 1-24
}

Response 201:
{
  "id": 12345,
  "checkout_url": "https://secure-checkout-url",
  "international_status": "WAITING_PAYMENT | PAID | DENIED | CANCELED | REFUNDED",
  "status": "string (pt-BR)",
  "installments": 3
}

Errors: 400 (invalid data), 401 (unauthorized), 500 (server error).

---

### 4. Create Cryptocurrency Payment

POST https://api-gateway.gates2b.com/payments/v2/crypto

NOTE: This endpoint uses a different base URL (api-gateway.gates2b.com).

Request body (JSON):
{
  "amount": 500.00,                        // required, fiat amount
  "fiat_currency": "BRL | USD | EUR",      // required
  "pay_currency": "USDTBSC | BTC | ETH",  // required
  "customer_name": "John Doe",             // required
  "customer_email": "john@example.com",    // required
  "external_reference": "INV-2024-001"     // required
}

Response 201:
{
  "id": "string",
  "id_transaction": "uuid",
  "status": "WAITING_PAYMENT",
  "payment_id": "string",
  "qr_code": "base64-encoded",
  "pay_address": "wallet-address?amount=X",
  "amount": 500.00,
  "currency": "BRL",
  "expires_at": "ISO 8601",
  "redirect_url": "string | null"
}

Error response:
{
  "error": "string",
  "message": "string",
  "details": "string",
  "statusCode": 400
}

### 5. Create Refund

POST /refunds/invoice/{invoiceId}

Refunds a payment (full or partial).

Path parameters:
  invoiceId (string, required): The invoice ID to refund.

Request body (JSON):
{
  "value": 50.00,              // optional, omit for full refund
  "reason": "Customer request" // optional
}

Response 201:
{
  "id": "string",
  "invoice": {
    "id": 12345,
    "transaction_pix_id": "string",
    "amount_final": 100.00,
    "status": "string"
  },
  "user": {
    "id": 1,
    "name": "string",
    "email": "string"
  },
  "gateway": "PIX | IUGU | ...",
  "transaction_id": "string",
  "amount": 50.00,
  "reason": "Customer request",
  "status": "REFUND_IN_PROGRESS | REFUND_COMPLETED | REFUND_FAILED | REFUND_CANCELLED",
  "external_refund_id": "string",
  "created_at": "ISO 8601",
  "updated_at": "ISO 8601"
}

Errors: 400 (invalid/insufficient balance), 401 (unauthorized), 404 (not found), 500 (error).

### 6. List Crypto Payment Invoices

GET /invoices/payment-gateway

Lists invoices with pagination and filtering.

Query parameters:
  page (number, default: 1)
  quantityPerPage (number, default: 10)
  orderBy (string, e.g. "createdAt.DESC")
  type (string: "PAYMENT" | "WITHDRAWAL")
  status (string)
  gateway (string)
  start_date (string, ISO 8601)
  end_date (string, ISO 8601)
  user_id (number, admin/operator only)
  query (string, text search)

Response 200:
{
  "status": 200,
  "code": "string",
  "response": {
    "data": [
      {
        "id": "string",
        "type": "PAYMENT",
        "status": "string",
        "amount": 100.00,
        "currency": "BRL",
        "gateway": "string",
        "user_id": 1,
        "external_reference": "string",
        "createdAt": "ISO 8601",
        "updatedAt": "ISO 8601",
        "payment_details": {
          "pay_address": "string",
          "qr_code": "string",
          "expires_at": "ISO 8601",
          "pay_currency": "USDTBSC",
          "fiat_currency": "BRL"
        }
      }
    ],
    "pagination": {
      "page": 1,
      "quantityPerPage": 10,
      "total": 100,
      "totalPages": 10
    }
  }
}

### 7. Get Crypto Payment Invoice Details

GET /invoices/payment-gateway/{id}

Path parameters:
  id (string, required): Invoice ID.

Response 200: Single invoice object (same structure as list endpoint item).

---

### 8. Resend Webhook

POST /invoices/resend-webhook/{id}

Manually triggers a webhook resend for a specific invoice.

Path parameters:
  id (string, required): Invoice ID.

Response 200:
{
  "message": "string",
  "invoice_id": 12345,
  "status": "success | pending"
}

Errors: 401 (unauthorized), 404 (not found), 500 (error).

## Webhooks

Configure your webhook URL at https://painel.gates2b.com → Integration section.
Webhooks fire when payment status changes (e.g., payment confirmed).

Webhook payload example (PAID):
{
  "id": 72276,
  "id_transaction": "uuid",
  "transaction_pix_id": "uuid",
  "external_reference": "string",
  "emv": "string",
  "amount": 120.00,
  "status": "Pagamento realizado",
  "international_status": "PAYMENT_RECEIVED",
  "name_client": "CUSTOMER NAME",
  "document_client": "00000000000",
  "createdAt": "ISO 8601",
  "my_origin": "Gates2b",
  "method": "Gates2b",
  "user": {}
}

International status values:
  WAITING_PAYMENT, PAYMENT_RECEIVED, PAID, DENIED, CANCELED, REFUNDED,
  REFUND_IN_PROGRESS, REFUND_COMPLETED, REFUND_FAILED, REFUND_CANCELLED

Retry policy: Up to 3 automatic retries with progressive intervals if your server
does not respond with HTTP 2xx. Use the resend webhook endpoint for manual retries.

## Quick Start Examples

### Node.js / TypeScript - Create PIX Payment

```typescript
const response = await fetch("https://api.gates2b.com/payments", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    clientMode: "noNeedToSpecify",
    amount: 49.90,
    expire_at: "2026-12-31T23:59:59Z",
    description: "Premium subscription",
    external_reference: "sub-001"
  })
});

const data = await response.json();
console.log(data.qr_image); // Base64 QR code to display
console.log(data.emv);      // Copy-paste PIX code
```

### Python - Create Crypto Payment

```python
import requests

response = requests.post(
    "https://api-gateway.gates2b.com/payments/v2/crypto",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_API_KEY"
    },
    json={
        "amount": 100.00,
        "fiat_currency": "BRL",
        "pay_currency": "USDTBSC",
        "customer_name": "John Doe",
        "customer_email": "john@example.com",
        "external_reference": "INV-2024-001"
    }
)

data = response.json()
print(data["pay_address"])  # Wallet address for payment
print(data["qr_code"])      # Base64 QR code
```

### cURL - Validate API Key

```bash
curl -X GET https://api.gates2b.com/api-key/validate \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### cURL - Create Refund

```bash
curl -X POST https://api.gates2b.com/refunds/invoice/12345 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"value": 50.00, "reason": "Customer request"}'
```

## Payment Status Flow

PIX: Created → WAITING_PAYMENT → PAYMENT_RECEIVED → (optional: REFUND_IN_PROGRESS → REFUND_COMPLETED)
Card: Created → WAITING_PAYMENT → PAID / DENIED → (optional: REFUNDED)
Crypto: Created → WAITING_PAYMENT → PAID → (optional: REFUNDED)

---

## Rate Limits & Best Practices

- Store your API key securely (environment variables, secrets manager).
- Always verify webhook signatures and validate payment amounts server-side.
- Use external_reference to correlate payments with your internal orders.
- Handle webhook retries idempotently (check if payment was already processed).
- Timestamps use America/Sao_Paulo timezone (UTC-3).
- Set appropriate expire_at values for your use case.
