---
title: "Criar Cobrança"
sidebarTitle: "POST /charge"
api: "POST https://api.gates2b.com/charge"
authMethod: "authorization"
description: "Cria uma cobrança. Pode opcionalmente criar a primeira tentativa de pagamento junto (PIX ou INTERNATIONAL) em um único request. Também é possível criar tentativas avulsas depois via POST /charge/{chargeId}/attempts."
---

<Info>
`paymentMethod = CREDIT_CARD` **não é aceito** neste endpoint — dados sensíveis de cartão devem passar apenas pelo checkout público (`POST /charge/{id}/checkout/payment` com `X-Checkout-Token`).
</Info>

<Note>
  <Badge color="green">Novo</Badge> **Split de Pagamento** — distribua o valor da cobrança entre múltiplos destinatários usando o campo `split[]`. Suporta valor fixo (`FIXED` em reais) e percentual (`PERCENT` sobre o `netAmount`). Veja os detalhes na seção de parâmetros abaixo.
</Note>

## Parâmetros

<ParamField header="Authorization" type="string" placeholder="Bearer {token}" required>
  Token de autenticação no formato Bearer.
</ParamField>

<ParamField body="grossAmount" type="string" placeholder="10.50" required>
  Valor bruto da cobrança em reais (duas casas decimais).
</ParamField>

<ParamField body="currency" type="string" placeholder="BRL" required>
  Moeda da cobrança (ISO 4217). Hoje sempre `BRL` — para pagamento em outra moeda, use `checkoutCurrency` com `paymentMethod = INTERNATIONAL`.
</ParamField>

<ParamField body="paymentMethod" type="string" placeholder="PIX">
  Se informado, cria a primeira tentativa junto com a cobrança.
  Valores aceitos:
  - `PIX` — cria tentativa com instruções PIX (QR code, EMV).
  - `INTERNATIONAL` — cria checkout session via Revolut. Requer `checkoutCurrency`.

  Se omitido, a cobrança é criada em `PENDING` sem tentativa.
</ParamField>

<ParamField body="checkoutCurrency" type="string" placeholder="USD">
  Moeda em que o pagador vai pagar no gateway (ISO 4217, ex.: `USD`, `EUR`, `MXN`). **Obrigatório** quando `paymentMethod = INTERNATIONAL`.
</ParamField>

<ParamField body="description" type="string" placeholder="Cobrança de teste">
  Descrição livre da cobrança.
</ParamField>

<ParamField body="expiresAt" type="string" placeholder="2030-12-31T23:59:59.000Z">
  Data de expiração no formato ISO 8601.
</ParamField>

<ParamField body="externalReference" type="string" placeholder="pedido-123">
  Referência externa do seu sistema para rastreio.
</ParamField>

<ParamField body="attemptIdempotencyKey" type="string" placeholder="uuid-v4">
  Chave de idempotência para a tentativa. Reuso da mesma chave retorna 409.
</ParamField>

<ParamField body="webhookUrl" type="string" placeholder="https://seu-site.com/webhooks/charge">
  URL chamada quando a cobrança for paga (POST com payload).
</ParamField>

<ParamField body="gatewayOptions" type="object">
  <Expandable title="Campos">
    <ParamField body="redirectUrl" type="string">URL de retorno após pagamento (Revolut).</ParamField>
    <ParamField body="expireAt" type="string">Expiração da tentativa no gateway (ISO 8601).</ParamField>
  </Expandable>
</ParamField>

<ParamField body="customerMeta" type="object">
  Dados do cliente. Usados pelos gateways quando não forem substituídos no momento do pagamento.
  <Expandable title="Campos">
    <ParamField body="name" type="string">Nome do cliente.</ParamField>
    <ParamField body="email" type="string">E-mail do cliente.</ParamField>
    <ParamField body="document" type="string">Documento (CPF/CNPJ).</ParamField>
    <ParamField body="document_type" type="string">Tipo do documento.</ParamField>
    <ParamField body="phone" type="string">Telefone.</ParamField>
    <ParamField body="source" type="string" placeholder="PRE_FILLED">Origem (`PRE_FILLED` ou `COLLECT_AT_CHECKOUT`).</ParamField>
  </Expandable>
</ParamField>

<ParamField body="split" type="array">
  <Badge color="green">Novo</Badge> Lista de destinatários para split de pagamento. Omitir este campo cria a cobrança sem split.
  <Expandable title="Campos de cada item">
    <ParamField body="email" type="string" required>
      E-mail do destinatário cadastrado na plataforma.

      <Warning>
        O destinatário precisa ter uma conta ativa na plataforma como usuário `CUSTOMER`. Emails não encontrados ou de usuários com outro perfil caem como residual para o dono da cobrança — **não bloqueia a criação**, mas o valor não será repassado ao destinatário pretendido.
      </Warning>
    </ParamField>
    <ParamField body="kind" type="string" required>
      Tipo do split:
      - `FIXED` — valor fixo em reais (informe `amount`). Soma dos itens FIXED deve ser ≤ `grossAmount`.
      - `PERCENT` — percentual sobre o `netAmount` (informe `percent`). Soma dos itens PERCENT deve ser ≤ 100.
    </ParamField>
    <ParamField body="amount" type="string" placeholder="50.00">
      Valor fixo em reais (ex.: `"50.00"`). **Obrigatório** quando `kind = FIXED`.
    </ParamField>
    <ParamField body="percent" type="number" placeholder="10">
      Percentual de 0.01 a 100. **Obrigatório** quando `kind = PERCENT`.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="id" type="string">
  Sem `paymentMethod`: ID da **cobrança**.
  Com `paymentMethod`: ID da **tentativa** (o ID da cobrança vai em `chargeId`).
</ResponseField>

<ResponseField name="chargeId" type="string">
  ID da cobrança (presente apenas quando a resposta é uma tentativa).
</ResponseField>

<ResponseField name="status" type="string">
  `PENDING` / `PROCESSING` / `PAID` / `EXPIRED` / `CANCELED` / `FAILED` / `REFUNDED`.
</ResponseField>

<ResponseField name="paymentMethod" type="string">
  `PIX` ou `INTERNATIONAL` (presente apenas quando há tentativa).
</ResponseField>

<ResponseField name="grossAmount" type="string">
  Valor bruto da cobrança (2 casas decimais).
</ResponseField>

<ResponseField name="currency" type="string">
  Moeda da cobrança (ISO 4217).
</ResponseField>

<ResponseField name="checkoutMeta" type="object">
  Instruções para o front concluir o pagamento.
  <Expandable title="Campos">
    <ResponseField name="url" type="string">URL pública do checkout.</ResponseField>
    <ResponseField name="paymentMethod" type="string">Método (`PIX` / `INTERNATIONAL`).</ResponseField>
    <ResponseField name="qr_image" type="string">Base64 da imagem QR (PIX apenas).</ResponseField>
    <ResponseField name="qr_text" type="string">BR Code PIX em texto (PIX apenas).</ResponseField>
    <ResponseField name="emv" type="string">EMV PIX (PIX apenas).</ResponseField>
    <ResponseField name="integrationToken" type="string">Token público para inicializar o Revolut Checkout Web SDK (INTERNATIONAL apenas).</ResponseField>
    <ResponseField name="currency" type="string">Moeda do checkout (INTERNATIONAL apenas).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fxMeta" type="object">
  Conversão cambial aplicada (INTERNATIONAL apenas).
  <Expandable title="Campos">
    <ResponseField name="fromCurrency" type="string">BRL.</ResponseField>
    <ResponseField name="toCurrency" type="string">Moeda destino (ex.: USD).</ResponseField>
    <ResponseField name="rate" type="number">Taxa aplicada.</ResponseField>
    <ResponseField name="originalAmount" type="number">Valor original em BRL.</ResponseField>
    <ResponseField name="convertedAmount" type="number">Valor convertido.</ResponseField>
    <ResponseField name="amountWithSpread" type="number">Valor final cobrado (com spread).</ResponseField>
    <ResponseField name="amountCents" type="number">Valor final em centavos.</ResponseField>
    <ResponseField name="fetchedAt" type="string">Momento da cotação.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="expiresAt" type="string">Data de expiração.</ResponseField>
<ResponseField name="description" type="string">Descrição da cobrança.</ResponseField>
<ResponseField name="externalReference" type="string">Referência externa.</ResponseField>
<ResponseField name="customerMeta" type="object">Dados do cliente.</ResponseField>
<ResponseField name="splitConfig" type="array">
  Configuração de split informada na criação (presente quando `split[]` foi enviado).
  <Expandable title="Campos">
    <ResponseField name="email" type="string">E-mail do destinatário.</ResponseField>
    <ResponseField name="kind" type="string">`FIXED` ou `PERCENT`.</ResponseField>
    <ResponseField name="amount" type="string">Valor fixo em reais (FIXED).</ResponseField>
    <ResponseField name="percent" type="number">Percentual (PERCENT).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="splitSettlement" type="array">
  Liquidação calculada: quanto cada destinatário recebe. O "troco" de arredondamento (PERCENT) sempre vai para o dono.
  <Expandable title="Campos">
    <ResponseField name="userId" type="number">ID do usuário na plataforma.</ResponseField>
    <ResponseField name="amountCents" type="number">Valor que o usuário receberá (centavos).</ResponseField>
    <ResponseField name="isOwner" type="boolean">`true` quando for o dono da cobrança.</ResponseField>
    <ResponseField name="sourceEmail" type="string">E-mail do destinatário.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">Data de criação (ISO 8601).</ResponseField>
<ResponseField name="updatedAt" type="string">Data de atualização (ISO 8601).</ResponseField>

## Criar Tentativa Avulsa

`POST /charge/{chargeId}/attempts`

Cria uma tentativa de pagamento para uma cobrança existente (server-side, autenticado via JWT). Aceita `PIX` e `INTERNATIONAL`.

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:-----------:|-----------|
| `paymentMethod` | string | Sim | `PIX` ou `INTERNATIONAL` |
| `currency` | string | Condicional | Moeda de destino (obrigatório para `INTERNATIONAL`, ISO 4217) |
| `redirectUrl` | string | Não | URL de retorno (usado em INTERNATIONAL) |
| `attemptIdempotencyKey` | string | Não | Chave de idempotência (recomendado) |

<CodeGroup>
```bash PIX
curl -X POST "https://api.gates2b.com/charge/{chargeId}/attempts" \
  -H "Authorization: Bearer {your-token}" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentMethod": "PIX",
    "attemptIdempotencyKey": "uuid-v4"
  }'
```

```bash INTERNATIONAL
curl -X POST "https://api.gates2b.com/charge/{chargeId}/attempts" \
  -H "Authorization: Bearer {your-token}" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentMethod": "INTERNATIONAL",
    "currency": "USD",
    "redirectUrl": "https://seu-site.com/retorno",
    "attemptIdempotencyKey": "uuid-v4"
  }'
```
</CodeGroup>
